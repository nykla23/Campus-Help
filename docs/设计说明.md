# **Campus-Help微信小程序设计说明文档** 

## 一、项目概述

### 1. 项目名称

校园帮

### 2. 项目定位

面向高校学生的校园互助平台，核心提供**取快递、代买饭、代送文件、学习辅导**等轻量级互助服务，解决学生校园内的生活与学习互助需求，打造 “简单、快速、低成本” 的学生互助体验。

### 3.用户研究

#### 用户群体

**核心用户**：本校在校大学生（本科生、研究生），年龄18-28岁，有校园生活需求。

**次要用户**：教职工（偶尔发布任务，如找人帮忙搬东西）、社团组织。

#### 用户场景

① 学生 A 在宿舍不想下楼，打开 APP 发布「取快递」跑腿任务，填写地址和赏金，等待接单；

② 学生 B 想赚零花钱，打开 APP 首页查看附近的跑腿任务，筛选「可接单」任务后点击抢单，完成后获取赏金；

③ 学生 C 完成跑腿任务后，打开个人中心查看收益，点击「提现」按钮完成账户提现，同时查看自己的接单记录。

### 3. 组件与布局规

#### （1）通用组件规范

- **按钮**：统一圆角（8px），主按钮采用实色填充（主色 #3EB1F4），次按钮采用描边 / 浅底样式；按钮与场景具体需求相适应，保证移动端点击舒适度。
- **输入框**：统一圆角（8px），边框采用浅灰描边。
- **卡片**：任务卡片 / 用户信息卡片采用白色底色 + 轻微阴影，任务卡片间距统一 16px，信息卡片间距统一 10px，保证视觉整洁与信息区分。
- **标签**：状态标签（如「待接单」）采用橙色实色填充，圆角 8px；功能标签（如「全部 / 待接单」）采用浅底 / 描边样式，选中态切换为主色填充。

#### （2）布局规范

所有页面遵循 **「顶部操作区 + 内容展示区 + 底部导航区」** 三层结构：

- 顶部操作区：固定高度，包含标题、搜索栏、返回按钮等核心操作；

- 内容展示区：独立垂直滚动，承载核心信息（任务列表、表单、详情等）；

- 底部导航区：固定高度（75px），包含「首页 / 发布 / 消息 / 我的」全局导航，不随内容滚动，保证全局操作便捷。

### 3. 首页（任务列表页）

#### （1）页面结构

- 顶部导航区：品牌标题「校园帮」+ 搜索栏 + 消息通知图标；
- 状态标签区：「全部 / 待接单 / 进行中 / 已完成」四个功能标签，横向排列，支持切换筛选；
- 筛选区：「筛选」按钮，点击展开任务类型标签（跑腿代办、取件代送、学习辅导、其他）；
- 任务列表区：垂直滚动的任务卡片，包含用户头像、信用分、任务标题、任务描述、虚拟币赏金、时间戳、状态标签；
- 底部导航区：「首页 / 发布 / 消息 / 我的」全局导航，当前选中「首页」高亮显示。

#### （2）任务卡片结构

- 左侧：用户头像 + 信用分（分数展示）；
- 中间：任务标题（如「代买午餐」）+ 任务描述（如「食堂→图书馆，12 点半送达」）+ 虚拟币赏金 + 截止时间提示；
- 右侧：状态标签（如「待接单」）。

#### （3）交互设计

- 点击「筛选」按钮：展开 / 收起类型标签组（原位展示，不居中），点击页面空白处可关闭标签组；
- 点击状态标签（全部 / 待接单等）：筛选对应任务列表，实时刷新内容；
- 点击任务卡片：跳转至**任务详情页**，查看任务完整信息；
- 点击底部导航图标：跳转至对应页面（首页 / 发布 / 消息 / 我的）。

- 表单区域：任务标题、详细描述、委托金额、地点（可选）、截止时间（可选）五个输入项，垂直排列；
- 提示区域：黑色边框提示框，展示「发布任务后，悬赏金额将暂时冻结。任务完成后，悬赏将自动转给接单者。任务取消后，悬赏将退还给您。」；
- 底部导航区：全局导航，当前选中「发布」高亮显示。

#### （2）交互设计

- 点击返回按钮：返回至**首页**；
- 点击任务类型标签：快速选择任务分类，选中态高亮；
- ~~点击输入框：触发输入状态，支持内容填写；~~
- 点击右上角「发布」按钮：提交任务信息，发布成功后跳转至首页并弹出提示；
- 点击底部导航图标：跳转至对应页面。

#### （3）设计亮点

- 任务类型快捷选择，降低用户填写门槛；
- 明确的金额冻结 / 退还规则提示，消除用户操作顾虑，提升信任度。

#### （2）交互设计

- 点击返回按钮：返回至**上一级页面**（如首页 / 我发布的页面）；

#### （3）设计亮点

#### （3）交互设计

#### （4）设计亮点

## 五、交互规范总结

1. **页面跳转**：所有页面跳转统一使用「Slide in from right」动画，贴合移动端操作习惯，跳转路径清晰（如按钮 / 卡片 / 导航图标触发）；
2. **筛选交互**：首页筛选标签采用「原位展示 / 收起」，避免居中遮挡，符合用户操作直觉；
3. **滚动规范**：所有内容区独立垂直滚动，顶部导航区与底部导航栏固定，不随内容滚动，保证全局操作便捷；
4. **反馈机制**：核心操作（如发布任务、接单、发送消息）均有明确的页面跳转 反馈，提升操作确定性。
5. **输入机制**：相关输入框尚未在figma上具体实现输入交互，仅为展示效果，其实际效果会在微信小程序上实现。
